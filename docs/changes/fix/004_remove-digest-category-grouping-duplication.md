# fix/004 移除 Digest 页面分类分组重复展示

## 事件概述
- 发现时间：2026-04-07
- 影响范围：`/digest` 页面列表展示
- 严重程度：中

## 时间线
- 2026-04-06 上一轮修复恢复了 Digest 页面按分类 section 分组。
- 2026-04-07 复查发现同一条内容带有多个分类时，会在多个 section 中重复渲染。
- 2026-04-07 确认当前产品需求是不分组的单时间线展示，因此移除页面层分类分组。

## 根因分析
直接原因是 `/digest` 页面在渲染前对记录执行了按 `categories` 的二次分组；多分类内容会被插入多个 section，从而重复显示。

根本原因是上一轮修复把“恢复分组”当成回归修复目标，但实际页面需求是保留后端排序后的单列表展示，没有重新核对当前信息架构约束。

## 修复方案
### 临时修复
移除 `/digest` 页面 `groupDigestRecords()` 分组逻辑，改为直接按 `getDailyDigestItems()` 返回顺序渲染单列表。

### 根本修复
- 页面层不再根据分类生成 section 标题与分隔线。
- 同步调整 loading skeleton，保持和单时间线布局一致。
- 保留服务端现有筛选与排序逻辑，仅修正前端展示重复问题。

## 预防措施
- [ ] 为 `/digest` 补充页面级测试，覆盖“多分类记录只渲染一次”的约束。
- [ ] 以后修复展示回归时，先确认当前产品需求，再决定是否恢复旧 UI 结构。
