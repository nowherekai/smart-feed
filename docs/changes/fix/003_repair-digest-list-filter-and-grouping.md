# fix/003 修复 Digest 列表筛选与分类分组回归

## 事件概述
- 发现时间：2026-04-06
- 影响范围：`/digest` 页面列表结果与分组展示
- 严重程度：中

## 时间线
- 2026-04-06 评审发现 `getDailyDigestItems()` 改为新筛选逻辑后，Digest 页面同步被改成扁平列表。
- 2026-04-06 确认页面不再按分类分组，多分类内容也失去跨分类展示能力。
- 2026-04-06 修复页面分组逻辑，并保留仅展示完整分析记录的服务端筛选条件。

## 根因分析
直接原因是 Digest 页面在重构时把原先的按分类分组逻辑删除，改成直接渲染扁平列表，导致分类信息没有进入最终展示结构。

根本原因是页面层展示重构和数据筛选调整同时发生，但没有覆盖“多分类内容需在每个分类 section 下展示”的已有行为约束。

## 修复方案
### 临时修复
恢复 `/digest` 页面服务端分组渲染，重新按分类输出 section 与分隔线结构。

### 根本修复
- `getDailyDigestItems()` 仅返回 `status="full"` 且存在摘要的记录，避免基础分析混入 Digest。
- 页面层恢复 `groupDigestRecords()`，让多分类记录在各自分类下可见。
- 对空分类记录归档到“未分类”，避免可渲染摘要被静默丢弃。

## 预防措施
- [ ] 后续为 `/digest` 增加覆盖多分类内容展示的页面级测试。
- [ ] 评审前优先检查“筛选逻辑变化 + 展示结构变化”是否同时影响既有信息架构。
