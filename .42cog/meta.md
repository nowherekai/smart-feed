# 项目元信息 (Meta)

## 基本信息

**项目名称**: smart-feed

**项目标语**: 将自定义订阅源转化为每日智能编排的个人情报摘要

**版本**: 0.1.0

**创建日期**: 2026-03-30

**负责人**: nowehrekai

## 项目描述

smart-feed 是一个基于用户自配置订阅源的个人信息情报系统，目标是把分散的信息源聚合、清洗、分析并生成每日 digest。
MVP 阶段聚焦 RSS，后续逐步扩展到 Podcast、Newsletter、微信公众号等来源。
项目强调可回链阅读、成本控制和可扩展的反馈闭环，让摘要结果保持可继续深读的入口。

## 目标用户

- 用户群体1: 需要高效消费多源信息的独立开发者、研究者和知识工作者
- 用户群体2: 希望基于自定义订阅源构建个人情报流并持续优化推荐偏好的用户

## 核心价值主张

1. 将分散且结构不一致的订阅内容统一清洗、标准化并沉淀为内部内容模型。
2. 使用 AI 对内容执行分类、关键词与实体抽取、单篇结构化摘要和日报编排，同时保留原文回链。
3. 通过分层处理、摘要缓存和后续可扩展的反馈闭环，持续降低成本并让输出越来越贴合个人需求。

## 技术栈

**前端**:
- 形态: Next.js Web 应用，负责来源配置、Digest 展示与后续扩展入口
- Web 框架: Next.js App Router
- UI: shadcn/ui + Tailwind CSS
- 语言: TypeScript

**后端**:
- 运行时: Bun
- 应用形态: Next.js API Routes + 后台 Worker
- 数据存储: PostgreSQL
- ORM: Drizzle ORM
- 任务队列: bullmq

**基础设施**:
- 配置管理: `.env` + Bun 原生环境变量加载
- 部署方式: Docker Compose / 任意支持 Docker 的平台
- 定时任务: bullmq 定时任务
- 运行模式: 单用户模式

**AI集成**:
- 核心能力: 分类、关键词抽取、实体识别、单篇摘要、digest 编排
- 模型策略: 轻模型做筛选与分类，重模型做总结与报告
- AI 适配层: Vercel AI SDK
- 模型供应商: 可切换，当前通过统一适配层接入
